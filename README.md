# Sistema Web para Pizzaria
`Documentação de Levantamento de Requisitos`   `Linguagem de Marcação`

Este repositório documenta a implementação de um protótipo de sistema web para uma pizzaria, desenvolvido como projeto integrador entre as Unidades Curriculares (UCs) de **Levantamento de Requisitos** e **Linguagem de Marcação**. O foco é a tradução de necessidades de negócio em uma arquitetura de software funcional, estruturada através de marcação semântica e estilização customizada.

## 📝 Escopo do Projeto
O desenvolvimento foi fundamentado na análise de processos de negócio típicos de uma pizzaria. O objetivo foi conceber uma solução que integrasse:
* **Gestão de Cadastros**: Clientes, estabelecimentos, usuários e formas de pagamento.
* **Experiência do Usuário (UX/UI)**: Fluxos de login, navegação via Landing Page e visualização do cardápio.
* **Identidade Visual**: Implementação de CSS customizado para garantir consistência em todas as telas criadas do zero.

## 🚀 Processo de Desenvolvimento
O projeto seguiu um fluxo estruturado, atendendo aos critérios de ambas as disciplinas:

1. **Elicitação e Requisitos**: Identificação dos atores (cliente, administrador) e dos fluxos de navegação necessários para o sistema.
2. **Semântica e Marcação**: Aplicação das boas práticas de **Linguagem de Marcação**, garantindo uma estrutura HTML5 limpa, acessível e organizada.
3. **Modelagem e Implementação**: Desenvolvimento das interfaces front-end, garantindo fidelidade ao design planejado e modularidade do código.

## 🛠️ Tecnologias e Metodologia
O projeto foi construído utilizando tecnologias fundamentais para o desenvolvimento web:
* **HTML5**: Estruturação semântica de todas as páginas, conforme as diretrizes da disciplina de Linguagem de Marcação.
* **CSS3**: Criação autoral de estilos, focada em design responsivo e identidade visual própria.
* **Engenharia de Requisitos**: Aplicação prática de conceitos de planejamento, desde a identificação de requisitos funcionais até a estruturação lógica do sistema.

## 📂 Organização Funcional
As telas foram organizadas para facilitar a manutenção e a escalabilidade, dividindo o sistema em módulos lógicos que representam as funcionalidades levantadas na fase de requisitos:

| Módulo | Objetivo na Regra de Negócio |
| :--- | :--- |
| **Login / Landing-Page** | Porta de entrada e autenticação do sistema. |
| **Cadastros (Cliente/User/Estab.)** | Gestão de entidades e persistência de dados. |
| **Cardápio** | Interface de exibição e interação com produtos. |
| **Configurações** | Suporte a estilos, fontes e ícones padronizados. |
